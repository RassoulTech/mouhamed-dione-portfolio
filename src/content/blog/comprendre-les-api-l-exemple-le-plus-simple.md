Une API, c'est partout dans le développement moderne. Et c'est bien plus simple que le mot le laisse croire.

## L'image du restaurant
Imagine un restaurant :
- **Toi** = ton application
- **La cuisine** = le serveur / la base de données
- **Le serveur (la personne)** = l'API

Tu ne vas pas en cuisine. Tu passes commande au serveur, qui transmet et te ramène ton plat. **L'API, c'est l'intermédiaire** qui transporte ta demande et la réponse.

## Concrètement
Une API permet à deux applications de **se parler**. Par exemple, une appli météo demande à l'API d'un service météo : « Quel temps à Dakar ? » et reçoit la réponse.

## Les mots à connaître
- **Requête** : ta demande
- **Réponse** : ce que l'API te renvoie (souvent du JSON)
- **Endpoint** : l'adresse précise où tu envoies ta demande

## Pourquoi c'est essentiel
Presque toutes les applis modernes utilisent des API : paiement, cartes, authentification, données… Comprendre les API, c'est comprendre comment le web communique.

> Une API, ce n'est pas de la magie. C'est juste un **messager** entre deux programmes. Une fois cette image en tête, tout devient clair.
