Tout développeur utilise Git. Bonne nouvelle : le concept est simple, même si le vocabulaire fait peur.

## Git, c'est quoi ?
Imagine un **bouton « sauvegarder »** ultra-puissant pour ton code. Chaque sauvegarde (un « commit ») garde une trace. Tu peux revenir en arrière à tout moment.

## GitHub, c'est quoi ?
C'est le **cloud** de ton code. Tu y stockes tes projets en ligne, tu les partages, tu collabores. C'est aussi ta vitrine de développeur.

## Les 4 commandes de base
- `git add` : « je prépare ces fichiers »
- `git commit` : « je sauvegarde avec un message »
- `git push` : « j'envoie sur GitHub »
- `git pull` : « je récupère les dernières versions »

## Pourquoi c'est vital
- Tu ne **perds jamais** ton travail
- Tu peux **revenir** à une version qui marchait
- Tu peux **collaborer** sans tout casser
- Ton **GitHub = ton portfolio** pour les recruteurs

> Git n'est pas une option. C'est le réflexe de base. Plus tôt tu l'adoptes, plus tôt tu codes comme un pro.
